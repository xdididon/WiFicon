About
WiFicon is an application for displaying relationships between devices on wireless networks.
Home Page
The main page contains 2 tables with clients and APs. The clients part is sorted by the highest count of probe requests recorded. The purpose of this table is to check and show on the map user's past associations. That is, if the user connected previously to WiFi-X and WiFi-Y, and location of these APs is in name.kismet.csv file, then display the APs on the map. The routers part of the table uses reversed logic. It displays MAC addresses of all clients whose probe requests match ESSID of particular AP. This table is also sorted by APs with the highest count of clients. Top of the list will usually have local businesses such as shopping centers or free WiFi hotspots. The list needs to be taken conservatively because WiFi names are ambiguous.
Plots Page
The plots available on this page show a breakdown of total number of probe requests for particular client during 24 hour time period. The data is pulled from the name.log.csv file but due to the file size, often in millions, it only shows clients who sent at least 200 probe requests in any given day. This way a vast majority of clients is ignored becuase the data would not yield any useful results - this is especially when only war-driving was used to obtain the network capture data. To get best results use static antenna to monitor specific area for longer time period (day/s). The plots should highlight time period of user's activity making it easier to spot habbits. The higher the number of probe requests in the plot (for one hour) the higher the likelihood of the device being active on internet. Since devices transmit probe requests periodically lower count would indicate that the device was present in the area but the user probably did not browse internet. Of course, the data relies on successfully capturing probe requests. The worst results were recorded when Airodump-ng was used in its default channel hopping options. This is bacause the antenna spends lots of time on channels with no or little clients.
Statistics Page
5x5 Grid:

The grid separates APs into different areas. This is accomplished by treating latitude and longitude co-ordinates as floats. The highests / lowest latitude and longitude values are pulled from the name.kismet.csv which form the edges of a rectangle - as pictured below. Width and height of the rectangle is simply defined as height = max lat - min lat and width = max lat - min lat. Both height and width is then divided by 5. This gives measures of each column / row in the grid. As not all cities and towns are square shaped the grid will include 0s along some of its edges. This should help to visualise the shape of the monitor area with the count of APs displaying the most populated areas.
